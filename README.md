# 软件测试课程作业 - 测试计划
### 1. 功能需求

本次测试的目标产品是一个模拟购物网站：`https://www.saucedemo.com`。
需要测试的核心功能点包括：

* **用户登录:**
    * 使用标准用户名和密码能成功登录。
    * 使用锁定的用户名不能登录，并显示正确的错误提示。
* **商品浏览与添加:**
    * 登录后，用户可以看到商品列表。
    * 用户可以将一件商品 "Sauce Labs Backpack" 添加到购物车。
* **购物车:**
    * 用户可以查看购物车，确认 "Sauce Labs Backpack" 已在购物车中。
* **登出:**
    * 用户可以成功从系统登出，返回登录页面。

### 2. 测试类型

* **功能测试:** 验证上述功能需求是否按预期工作。
* **回归测试 :**在Beta阶段，可以重复运行自动化脚本，以确保新的更改没有破坏现有功能。

### 3. 测试环境 (Test Environment)

* **操作系统:** Windows 11

* **自动化框架:** Selenium 4
